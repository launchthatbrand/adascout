This template uses Next.js parallel routes to inject layout sections into a
single root shell.

## Slot mapping

- `src/app/@header/default.tsx` -> header slot
- `src/app/@footer/default.tsx` -> footer slot
- `src/app/@sidebar/default.tsx` -> sidebar slot

`src/app/layout.tsx` receives these as `props.header`, `props.footer`, and
`props.sidebar`.

<Callout title="Why this matters" type="info">
  Teams can swap header/footer/sidebar behavior per app while keeping a shared
  layout contract.
</Callout>
